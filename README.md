# Finance Dashboard & Management System

This repository contains a full-stack web application built to help teams manage, track, and visualize their financial records. We designed this platform to handle everything from user permissions down to generating visual expense reports, all while keeping the user interface clean and responsive.

---

## Key Features

### Role-Based Access Control
Security and user permissions are handled through a stateless JWT authentication system. We've set up three distinct roles to govern what users can and cannot do:
- **Admin**: Has full control over the system. Admins can manage all financial records, change user roles, and activate or deactivate accounts.
- **Analyst**: Can view the dashboard analytics and read all financial records to monitor performance, but they cannot make modifications to the data.
- **Viewer**: Has basic read-only access to see general records, without any administrative privileges.

### Financial Records Management
The core of the application revolves around tracking income and expenses. Users with the appropriate permissions can create, update, and delete transaction records. Since data tables can get large quickly, we implemented server-side pagination and a powerful filtering engine. You can easily filter the data by date ranges, specific categories like "Rent" or "Salary", transaction types, or just use a standard text search to find what you need.

### Dashboard Analytics
To give users an immediate sense of financial health, the dashboard aggregates all the raw data into visual formats. It calculates the net balance, total income, and total expenses in real-time. We used Recharts to plot income versus expense trends over time and to provide pie chart breakdowns of how money is being spent across different categories. There is also a quick-access feed showing the most recent transactions right on the front page.

### User Management Console
For administrators, we built a dedicated panel to manage the people using the software. Admins can view a list of all registered users, dynamically upgrade or downgrade their roles, and softly suspend accounts when necessary to cut off access, all without needing to dive into the database directly.

## Technology Stack

**Frontend Interface**
The frontend is a single-page application built with React and bundled using Vite for speed. We used Tailwind CSS to create a modern, responsive layout that works well on both desktop and mobile devices. All network requests are routed through Axios, which automatically intercepts API calls to attach the necessary authorization tokens.

**Backend Architecture**
The backend operates on Java 17 and Spring Boot 3.2. We configured Spring Security to manually handle JWT filters and manage stateless sessions across the API. Data persistence is managed by Hibernate/JPA and connected to a remote Supabase PostgreSQL database. 

During development, we also structured the SQL queries explicitly to avoid common parameter-binding issues that sometimes occur across different compiling environments.

## Getting Started

If you want to run this project locally, make sure you have Node.js, Java 17+, and Maven installed on your machine.

### 1. Setting up the Backend
By default, the backend server will start on port 8080. We built a DataSeeder into the application, so if it detects an empty database on startup, it will automatically populate some sample users and financial records for you.

To start the server, open your terminal and run:
```bash
cd backend
mvn clean install
mvn spring-boot:run
```

### 2. Setting up the Frontend
The frontend runs on port 5173. We've configured Vite to automatically proxy API requests to your local backend, so you shouldn't have to worry about CORS issues while testing.

Open a new terminal window and run:
```bash
cd frontend
npm install
npm run dev
```

### 3. Default Login
Once everything is running, you can log into the application using the admin account generated by the seeder:
- **Username:** admin
- **Password:** password123
